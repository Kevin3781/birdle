Run the Birdle automated integration tests and surface the results.

## Steps

1. Run the test suite:
   ```
   cd C:/src/birdle && python test_birdle.py
   ```

2. Print the full test output so the user can see pass/fail for each of the 3 tests.

3. Send the following screenshots to the user using SendUserFile (they are always regenerated by the test):
   - `C:/src/birdle/screenshots/03_daily_initial.png` — game on load
   - `C:/src/birdle/screenshots/04_after_wrong_guess.png` — after a wrong guess with hint
   - `C:/src/birdle/screenshots/05_daily_bird_card.png` — bird reveal card
   - `C:/src/birdle/screenshots/07_practice_game_start.png` — practice mode
   - `C:/src/birdle/screenshots/08_practice_bird_card.png` — practice result card

   If any FAIL_*.png screenshots exist in `C:/src/birdle/screenshots/`, send those too and note which test failed.

4. Report a one-line summary: how many passed, how many failed, and any JS console errors.

## Context

- HTTP server runs on port 8743, serving `C:/src/birdle`
- The test script starts and stops the server automatically
- `DAILY_BIRD` in the test must match the actual daily bird for day 528 (2026-06-13) from `getDailyBird()` in `birds.js`. If daily mode test fails with "Wrong bird on card", recalculate: `day 528 % GLOBAL_POOL.length` and update `DAILY_BIRD` in `test_birdle.py`.
- The `add_init_script` suppresses the first-run help modal so it doesn't block test interactions
- Practice mode test uses 5 London-region bird guesses; it passes whether the practice bird is guessed correctly or via game-over after 5 wrong guesses
